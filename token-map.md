# Henri Lloyd — Token Map
## Design token → Stripo output reference
**Active modes: Dark Base · Desktop (primary) · Mobile (override layer)**

This file is the single reference for translating Figma local variables into Stripo email CSS and HTML.
All alias references (`{spacing.L}`, etc.) are resolved to their literal values.
**Never use CSS custom properties (`var()`) in email output — email clients strip them.**

---

## How to read this document

| Column | Meaning |
|---|---|
| Token path | Dot-separated path inside the JSON collection |
| Resolved value | Literal value after alias resolution |
| Stripo layer | **Default CSS** · **Custom CSS** · **Inline** |
| CSS target | The selector or HTML element that receives the value |
| Property | The CSS property name |

### The three Stripo layers

| Layer | Location in Stripo UI | What belongs here |
|---|---|---|
| **Default CSS** | Appearance → General Styles | **Read-only.** Generated by the General Styles panel (Default Styles = ON). Do not edit or replace it. |
| **Custom CSS** | Custom CSS panel | **All our token-driven overrides live here**: h1–h6 scale, body/link/list styles, colours, `@media` mobile blocks. Override Default CSS via normal cascade. |
| **Inline** | `style=""` on the HTML element | Block-specific padding, per-block background, one-off font exceptions |

**Cascade rule:** Inline always wins over Default/Custom CSS without `!important`. The only place `!important` is required is inside `@media` mobile blocks, to override desktop inline styles.

## 3. Typography tokens — Desktop

Source: `Figma-local-variables/Typography Tokens - New/Desktop.tokens.json`
Active mode: **Desktop**
Layer: **Default CSS** for all heading and body rules. Inline only for button instance properties.

> **Line-height rule:** Always express as absolute px. Never %, never unitless — Outlook.com ignores them.

### 3.1 Headings

| Element | font-family | font-size | line-height | font-weight | CSS weight | letter-spacing | margin-bottom |
|---|---|---|---|---|---|---|---|
| `h1` | DM Sans | `48px` | `48px` | Regular | `400` | `0` | `24px` (from `spacing.text.h1-pad-bottom`) |
| `h2` | DM Sans | `32px` | `32px` | Semibold | `600` | `0` | `20px` |
| `h3` | DM Sans | `24px` | `24px` | Semibold | `600` | `0` | `16px` |
| `h4` | DM Sans | `20px` | `20px` | Semibold | `600` | `0` | `12px` |
| `h5` | DM Sans | `15px` | `18px` | Bold | `700` | `0` | `10px` |
| `h6` | PT Mono | `12px` | `12px` | Regular | `400` | `0` | — |

**h1 font-weight:** Confirmed `Regular (400)`. The typography token explicitly overrides the Brand `Semibold` default. This is intentional for Dark Base — h1 is a display/hero heading.

**h6 font-family:** `PT Mono` — confirmed used in the reference HTML (product label/SKU lines). A separate font link for PT Mono must be added to `<head>` in any template that uses h6.

### 3.2 Body and utility styles

## 9. Known constraints

| Constraint | Detail |
|---|---|
| **`es-text-{id}` mobile selectors** | Stripo assigns these IDs at import time. Leave mobile font-size block overrides as commented placeholders in Custom CSS; fill in after import. |
| **PT Mono (h6)** | Not included in the Shopify CDN DM Sans link. Add a separate font link or `@font-face` for PT Mono in any template that uses `h6`. |
| **Letter-spacing on Outlook desktop** | `letter-spacing` renders inconsistently in Outlook for Windows. Our heading letter-spacing is `0` (no override), so this is a non-issue for this system. |
| **Button border-radius on Outlook desktop** | `border-radius` is silently ignored by Outlook desktop. The `es-button-border` + `es-button` pattern handles this gracefully — Stripo injects VML at compile time. |
| **`margin-bottom` on `<p>` and `<h>`** | Use capital `Margin` in Outlook-targeted rules (`.es-wrapper, body { Margin: 0 }`). For per-element bottom spacing, `margin-bottom` on `<h>` elements is safe; `margin-bottom` on `<p>` should be tested in Outlook. |
| **Spacer blocks** | Not used as a default spacing mechanism. Reserved as developer wildcards for one-off custom spacing needs within a template. |
| **`.es-header-body`** | Not present in the reference template, but valid. Include its colour rules in Default CSS — they activate whenever a header stripe is added. |
| **Mobile stripe content-width (360px)** | Figma canvas reference only. Produces no CSS output. |
